# Skill: GBP SEO 商家檔案分析報告產生器

## 觸發時機

當使用者提供商家資訊（Google Maps 連結、店家名稱或地址），要求產生 GBP SEO 健檢報告時使用此流程。

## 使用者會提供

- **商家資訊**（至少一項）：Google Maps 連結、店家名稱、地址
- **Email**：報告寄送對象
- **稱呼**：報告中的收件人稱呼

## 分析流程

### 1. 抓取商家檔案資料

根據提供的商家資訊，蒐集以下資料：

- 商家名稱、地址、電話、營業時間
- 商家類別（主類別 + 次要類別）
- 商家描述
- 評論數量、平均評分、近期評論內容
- 商家回覆評論的狀況（有無回覆、回覆率）
- 照片數量、類型（店面/產品/團隊）
- Google 貼文（有無、頻率）
- 商家屬性與服務項目

### 2. 四大面向評分

每個面向滿分 100 分，總分為四個面向的加權平均。

#### 面向一：商家基本資訊完整度（權重 25%）

| 檢查項目 | 配分 | 說明 |
|---------|------|------|
| 商家名稱 | 10 | 是否包含關鍵字、是否過度堆砌 |
| 地址 | 10 | 是否完整、是否與 Google 地圖一致 |
| 電話 | 10 | 是否有填寫、是否為本地號碼 |
| 營業時間 | 15 | 是否完整、是否標示特殊營業時間 |
| 商家類別 | 15 | 主類別是否正確、是否有設定次要類別 |
| 網站連結 | 10 | 是否有填寫、是否可正常連線 |
| 商家屬性 | 15 | 付款方式、無障礙設施等屬性填寫率 |
| 服務項目/菜單 | 15 | 是否有建立產品/服務清單 |

#### 面向二：評論分析（權重 30%）

| 檢查項目 | 配分 | 說明 |
|---------|------|------|
| 評論總數 | 20 | 與同業比較的評論量級 |
| 平均評分 | 20 | 4.0 以上為佳，4.5 以上為優 |
| 近期評論 | 20 | 最近 30 天是否有新評論 |
| 負評處理 | 20 | 負面評論的回覆率與回覆品質 |
| 整體回覆率 | 20 | 所有評論的回覆比例 |

#### 面向三：照片與視覺內容（權重 20%）

| 檢查項目 | 配分 | 說明 |
|---------|------|------|
| 照片總數 | 25 | 至少 10 張以上為基本 |
| 照片多樣性 | 25 | 是否涵蓋店面、內裝、產品、團隊 |
| 封面照與 Logo | 25 | 是否有設定、品質是否專業 |
| 更新頻率 | 25 | 是否持續上傳新照片 |

#### 面向四：描述與關鍵字策略（權重 25%）

| 檢查項目 | 配分 | 說明 |
|---------|------|------|
| 商家描述 | 30 | 是否有撰寫、長度是否充足（150 字以上） |
| 關鍵字覆蓋 | 30 | 描述中是否包含目標關鍵字 |
| Google 貼文 | 20 | 是否有發布、頻率如何 |
| Q&A 區塊 | 20 | 是否有主動建立常見問答 |

### 3. 產生優化建議

針對每個面向中扣分的項目，提供：
- **問題描述**：具體說明目前的問題
- **影響程度**：高 / 中 / 低
- **具體步驟**：逐步說明如何改善（要到「照著做就能完成」的程度）
- **預估效果**：改善後預期的分數提升

### 4. 輸出報告 HTML

報告結構如下：

```
┌─────────────────────────────────┐
│  報告標題                        │
│  商家名稱 — GBP SEO 健檢報告     │
│  產生日期：YYYY-MM-DD            │
├─────────────────────────────────┤
│  總分圓環圖（0-100）             │
│  四大面向分數摘要                │
├─────────────────────────────────┤
│  面向一：商家基本資訊完整度       │
│  - 分數 + 各項目得分             │
│  - 問題列表 + 優化步驟           │
├─────────────────────────────────┤
│  面向二：評論分析                │
│  - 分數 + 各項目得分             │
│  - 問題列表 + 優化步驟           │
├─────────────────────────────────┤
│  ★ 中段 CTA                    │
│  「想了解如何運用 AI 分析         │
│   你的商家？」                   │
│  → 預約 1hr AI 諮詢 NT$4,000   │
│  → 連結導向 Potaly 官網         │
├─────────────────────────────────┤
│  面向三：照片與視覺內容          │
│  - 分數 + 各項目得分             │
│  - 問題列表 + 優化步驟           │
├─────────────────────────────────┤
│  面向四：描述與關鍵字策略         │
│  - 分數 + 各項目得分             │
│  - 問題列表 + 優化步驟           │
├─────────────────────────────────┤
│  優化行動清單（按優先順序）       │
│  ☐ 第 1 步：...                 │
│  ☐ 第 2 步：...                 │
│  ☐ 第 3 步：...                 │
├─────────────────────────────────┤
│  ★ 底部預告                     │
│  「14 天後我們會回來看看          │
│   你的進步！」                   │
└─────────────────────────────────┘
```

### 5. 儲存報告

- 檔案命名：`data/YYYYMMDD-slug.html`
- slug 從商家名稱取英文部分，全小寫，用 `-` 連接
- 範例：`data/20260307-zarco-store.html`

### 6. 註冊到 projects.json

使用 `add-project` skill 的流程，將報告加入 `data/projects.json` 並執行 `python3 build.py`。

## 報告設計原則

1. **完整不閹割** — 免費報告要真心幫用戶提升曝光度，不刻意留白
2. **具體可執行** — 每個優化建議都要到「照著做就能完成」的程度
3. **語氣友善** — 不用專業術語嚇人，用商家老闆能理解的語言
4. **CTA 自然嵌入** — 中段 AI 諮詢 CTA 和底部 14 天跟進預告要融入報告脈絡，不突兀

## 評分顏色規則

| 分數區間 | 顏色 | 狀態 |
|---------|------|------|
| 80-100 | 綠色 #34A853 | 優秀 |
| 60-79 | 藍色 #4285F4 | 良好 |
| 40-59 | 黃色 #F9AB00 | 待改善 |
| 0-39 | 紅色 #EA4335 | 急需改善 |

## 14 天跟進報告（Before / After）

當 n8n 排程觸發 14 天跟進時，重新執行相同分析流程，並額外產出：

- **對比表格**：14 天前 vs. 現在的各項分數
- **改善標記**：綠色箭頭（↑ 改善）/ 紅色箭頭（↓ 退步）/ 灰色橫線（— 無變化）
- **跟進 CTA**：
  - 有改善 → 「繼續保持！想每月追蹤進度嗎？」→ 訂閱月度追蹤
  - 沒改善 → 「需要幫忙嗎？」→ 預約免費 15 分鐘諮詢
